# GrowthBook Provider

The GrowthBook provider lets you manage [GrowthBook](https://www.growthbook.io)
feature flagging, experimentation, and analytics configuration as code. It targets
the GrowthBook REST API (`/api/v1`) and works with both GrowthBook Cloud and
self-hosted instances.

## Features

- **Feature flags** — full value type, default value, and per-environment
  enablement with force / rollout / experiment / experiment-ref targeting rules.
- **Targeting** — attributes, saved groups, and namespaces.
- **Metrics & data modeling** — metrics, metric groups, fact tables and filters,
  fact metrics, dimensions, and segments.
- **Experimentation & governance** — experiment and ramp-schedule templates,
  archetypes, custom fields, teams, and dashboards.
- **Data sources** — read existing resources, plus read-only organization members.
- **Import support** for all resources.

## Requirements

- [Terraform](https://www.terraform.io/downloads.html) >= 1.0 (or OpenTofu >= 1.6)
- A GrowthBook account (Cloud or self-hosted) with API access
- A GrowthBook API key with appropriate permissions

## Getting your GrowthBook API key

1. Log in to your GrowthBook instance.
2. Go to **Settings** → **API Keys** (or **Personal Access Tokens**).
3. Create a key and copy it for use in the provider configuration.

## Example Usage

```terraform
terraform {
  required_providers {
    growthbook = {
      source  = "pinotelio/growthbook"
      version = "~> 0.1"
    }
  }
}

# Credentials can also be supplied via GROWTHBOOK_API_KEY / GROWTHBOOK_API_URL.
provider "growthbook" {
  api_key = var.growthbook_api_key
  api_url = "https://api.growthbook.io/api/v1"
}
```

## Environment Variables

The provider can also be configured via environment variables:

```bash
export GROWTHBOOK_API_KEY="secret_..."
export GROWTHBOOK_API_URL="https://api.growthbook.io/api/v1"
```

## Authentication

Provide your credentials in one of two ways:

1. **Direct configuration** — set `api_key` (and optionally `api_url`) in the
   `provider "growthbook"` block.
2. **Environment variables** — set `GROWTHBOOK_API_KEY` and `GROWTHBOOK_API_URL`.

Store the API key securely (environment variables or a secrets manager); never
commit it to version control.
